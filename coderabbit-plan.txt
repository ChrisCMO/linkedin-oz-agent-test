**Task 4.1a — `services/ai_svc.py` (extend with `score_company`):**
- Add `score_company(company_data, icp_config) → CompanyScoringResult` function
- **Model routing:** GPT-5.4 for scoring operations
- **Eight weighted dimensions:**
  - Industry Fit (20 pts): match against `icp.industry_priorities` with explicit sub-category mappings
  - Revenue Fit (15 pts): compare against `icp.revenue_sweet_spot`; bonus for sweet spot center, penalty approaching ceiling
  - Company Size/Employees (15 pts): compare against `icp.employee_threshold_trigger` and ideal range
  - Geography (12 pts): match against `icp.geography_tiers`; tier 1 = full points, tier 2 = partial
  - Ownership Type (10 pts): clean = full points; PE-backed = 0 + flag; excluded types = hard disqualify
  - Growth Signals (10 pts): detect from Apollo/ZoomInfo data (recent funding, headcount growth >20%, new offices, M&A)
  - Leadership Change (10 pts): new CFO/Controller in past 12 months = +10 (audit relationship reset opportunity)
  - Competitor Signal (8 pts): detect current auditor is Moss Adams/BDO/Sweeney Conrad from enrichment or connection analysis
- **Competitor Signal constraint (enforce in prompt):** bolster-only — adds points only if base score (without Competitor Signal) is ≥50; a company scoring <50 on other dimensions cannot be qualified solely by Competitor Signal
- **Score routing thresholds:** ≥70 = Qualify; 50–69 = Review; <50 = Skip
- Scoring prompt must include: dimension definitions and weights, explicit sub-category industry mappings (e.g., "civil engineering" → Construction), model client calibration anchors from Phase 0.5.2, Competitor Signal constraint enforcement
- Return `CompanyScoringResult` with score, dimension_breakdown, reasoning, flags (pe_backed, competitor_detected, leadership_change, growth_detected)

**Task 4.1b — `services/ai_svc.py` (extend with `score_contact`):**
- Add `score_contact(contact_data, company_context) → ContactScoringResult` function
- **Model routing:** GPT-5.4 for scoring operations
- **Five dimensions:**
  - Title Match (30 pts): exact match to `icp.title_targeting` = 30; related = 20; tangential = 10
  - Seniority Level (25 pts): C-suite = 25; VP = 20; Director = 15; Manager = 10
  - LinkedIn Availability (20 pts): URL present = 20; missing = 0
  - Tenure at Company (15 pts): 1–3 years = 15; <1 year = 10; >5 years = 5
  - Email/Phone Availability (10 pts): both = 10; one = 5; neither = 0
- **NOT a qualification gate:** Contact score is used only for intra-company contact prioritization; it never overrides Company ICP Score qualification decisions
- Return `ContactScoringResult` with score, dimension_breakdown, reasoning

**Task 6.1 — `lib/unipile.py` (safety-critical wrapper):**
- Create as a peer to `lib/apollo.py`, `lib/zoominfo.py`, `lib/google_places.py`, `lib/apify.py` — not an ad hoc utility; direct Unipile API access from skills or services is explicitly prohibited
- Implement functions: `send_connection_request(account_id, linkedin_url, note) → SendResult`, `send_message(chat_id, text) → SendResult`, `get_conversation(chat_id) → Conversation`, `get_relations(account_id) → List[Relation]`, `get_profile(linkedin_url) → Profile`
- **Pre-flight safety guard on every write operation** (`send_connection_request`, `send_message`): six sequential checks; any failure aborts before the call reaches Unipile: (1) account `status: 'active'` in `linkedin_accounts`; (2) `check_rate_limit(account, action)` counts today's rows in `activity_log`; (3) `is_business_hours(account)` — 8 AM–6 PM tenant timezone, no weekend connections; (4) `provider_config.cooldown_until` has passed; (5) prospect not already 1st-degree connection; (6) no pending invitation in `invitations` table
- **Activity logging as authoritative rate limit mechanism:** log every API call to `activity_log` (partitioned quarterly) with `action_type`, `account_id`, `created_at` — source of truth for `check_rate_limit()` counts
- **Random delay injection (mandatory, non-bypassable):** 45–120 second random delay between consecutive LinkedIn actions per account; bot fingerprint prevention; cannot be disabled or overridden
- **Profile lookup enforcement:** all `get_profile()` calls MUST include `linkedin_sections=*` query param
- **Chat ID management:** track `chat_id` per prospect in `prospects.sequence.chat_id` JSONB; capture `chat_id` from first message response and persist to ensure follow-ups route to correct thread
- **Wrapper responsibilities** (NOT delegated to Oz): auth/header construction (DSN/API key from `linkedin_accounts.provider_config`), retries/backoff (exponential with jitter on 429/5xx), response normalization, circuit breaker (on 422/429, pause all account actions and set `provider_config.cooldown_until`)
